## Context

The [OpenSSF Best Practices Badge](https://www.bestpractices.dev/en) is a voluntary self-certification
from the Open Source Security Foundation (Linux Foundation). Projects demonstrate adherence to 67
security, quality, and governance criteria across six categories. Earning the **passing** badge signals
to users, integrators, and procurement teams that the project follows industry-recognized practices.

The project already meets **~90% of criteria** thanks to existing CI/CD, testing, documentation, and
security infrastructure. This epic closes the remaining gaps and tracks the badge submission.

## User Story

> "As a project maintainer, I need an externally recognized security and quality badge so that
> public-sector procurement teams and open-source integrators can trust our engineering practices
> without requiring a custom audit."

---

## Goals

1. Close all MUST-level gaps so the project qualifies for the OpenSSF passing badge.
2. Document the full 67-criterion audit as a living reference for ongoing compliance.
3. Establish a release process that satisfies change-control criteria (tags, release notes, CVE tracking).
4. Submit the badge application and display the badge in README.md.

---

## Out of Scope

- Silver or Gold badge levels (future epic).
- Additional DAST hardening beyond the nightly OWASP ZAP baseline scan (criterion #64 is now MET).
- Automated CVE scanning beyond existing Dependabot + pip-audit + npm-audit setup.

## MVP Scope

### 1) Governance documentation

- Create `CODE_OF_CONDUCT.md` at repo root (Contributor Covenant 2.1). Not a badge criterion itself
  but expected for open-source projects and strengthens the `contribution` criterion answer.

### 2) Release process

- Add a **Releases** section to `CONTRIBUTING.md` documenting:
  - Git tag convention: `v{MAJOR}.{MINOR}.{PATCH}` (SemVer)
  - CHANGELOG.md format: [Keep a Changelog](https://keepachangelog.com/) headers above CI entries
  - Security section in release notes: list CVE IDs for any fixed vulnerabilities
- Restructure `CHANGELOG.md` top section with feature-oriented format template.

### 3) Badge submission

- Self-certify at [bestpractices.dev](https://www.bestpractices.dev/en) using audit tables above.
- Add badge SVG to `README.md`.
