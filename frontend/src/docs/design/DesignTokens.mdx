# Design Tokens

Terminandoyo uses a **Paperclip Design** language: paper-like surfaces, blueprint+industrial accents,
subtle motion (lift, snap, unfold), and structure-revealing visuals.

All tokens are defined as CSS custom properties in `src/index.css` inside the `@theme` block,
making them available as Tailwind utility classes (e.g. `bg-paper-100`, `text-blueprint-500`).

---

## Status (Feedback)

Semantic feedback colors for errors, success, warnings, and informational messages.
Each token is a single base color — use Tailwind opacity modifiers for tinted backgrounds
(`bg-status-error/10`) and borders (`border-status-error/30`), following the same
`/15` tint pattern as bucket badges.

| Token                    | Value / Maps to            | Usage                                       |
| ------------------------ | -------------------------- | ------------------------------------------- |
| `--color-status-error`   | `oklch(0.55 0.20 25)`     | Error text, icons, destructive buttons      |
| `--color-status-success` | `oklch(0.50 0.16 145)`    | Success text, icons, completion indicators  |
| `--color-status-warning` | `oklch(0.52 0.14 55)`     | Warning text, icons, caution banners        |
| `--color-status-info`    | `var(--color-blueprint-500)` | Informational text, icons (matches primary) |

**Usage patterns:**

```
text-status-error                    /* icon / label */
bg-status-error/10                   /* subtle background */
border-status-error/30               /* container border */
bg-status-error text-white           /* solid destructive button */
```

## Semantic Aliases

Higher-level tokens that map to the raw palettes above. **Prefer these in components** for
consistency — if the primary color changes from blueprint to something else, only the aliases
need updating.

| ------------------------ | ------------------------------ | ---------------------------- |
| `--color-surface`        | `var(--color-paper-50)`        | Page background              |
| `--color-surface-raised` | `var(--color-paper-100)`       | Cards, raised panels         |
| `--color-surface-overlay`| `var(--color-paper-200)`       | Overlays, dropdowns          |
| `--color-border`         | `var(--color-paper-300)`       | Default borders              |
| `--color-border-strong`  | `var(--color-paper-400)`       | Emphasized borders           |
| `--color-text`           | `oklch(0.25 0.01 260)`        | Body text                    |

| `--color-text-subtle`    | `oklch(0.43 0.008 260)`       | Tertiary / hint text         |
| `--color-primary`        | `var(--color-blueprint-600)`   | Primary actions, buttons     |
| `--color-primary-hover`  | `var(--color-blueprint-700)`   | Primary hover state          |
| `--color-accent`         | `var(--color-industrial-500)`  | Accent actions, highlights   |

## Motion

Subtle, purposeful animations inspired by physical paper interactions.

| Token                | Value                            | Usage                     |
| -------------------- | -------------------------------- | ------------------------- |
| `--duration-instant` | 100ms                            | Micro-feedback            |
| `--duration-fast`    | 150ms                            | Button press, chip toggle |
| `--duration-normal`  | 250ms                            | Card transitions          |
| `--duration-slow`    | 400ms                            | Panel open/close          |
| `--ease-snap`        | `cubic-bezier(0.2, 0.8, 0.2, 1)` | Attach/detach snap        |
| `--ease-unfold`      | `cubic-bezier(0.4, 0, 0.2, 1)`   | Panel unfold              |
