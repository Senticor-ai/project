# Secrets in Documentation

Guidelines for documenting example credentials without triggering secret scanners
(gitleaks, detect-secrets, bandit).

---

## Placeholder Patterns (Preferred)

Use placeholder patterns that scanners automatically recognize as non-secrets.
No suppression markers needed:

```bash
# Angle brackets
DB_PASSWORD=<your-db-password>

# Shell variable syntax
API_KEY=${API_KEY}

# Template syntax
TOKEN={{TOKEN}}

# Obvious placeholders
SECRET=<SECRET>
```

### Common auto-ignored patterns

| Pattern | Example |
|---------|---------|
| Angle brackets | `<your-password>`, `<DB_PASSWORD>` |
| Shell variables | `${VAR}`, `$VAR` |
| Template syntax | `{{VAR}}` |
| Placeholder words | `your_api_key_here`, `changeme`, `placeholder` |

---

## Fallback Markers

Only use suppression markers when placeholder patterns are not possible:

| Scanner | Marker |
|---------|--------|
| gitleaks | `# gitleaks:allow` |
| detect-secrets | `# pragma: allowlist secret` |
| bandit (Python) | `# nosec B105` |

---

## Core Rule

**Never put real passwords, API keys, or secrets directly in code, config files, or
`docker-compose.yml`.** Use environment variables with `${VAR:-default}` syntax for
local development defaults.
