## Filter Behavior

- **Context chips**: multi-select OR logic (show actions matching ANY selected context)
- **Energy**: single-select, filters by `ComputationPort.energyLevel`
- **Time**: "I have X minutes" â€” shows actions with `timeEstimate <= X`
- **Tab + filters combine**: tab sets the base query, filters narrow it
- **URL-synced**: filter state reflects in URL for deep linking
